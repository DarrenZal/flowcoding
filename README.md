# Flow Coding

**The evolution of AI-assisted development — from vibe to craft.**

→ [flowcode.ing](https://flowcode.ing)

---

Vibe coding showed us that anyone can build with AI. Flow coding is what happens when you turn that into an art form — a practice of deep focus, creative intuition, and sustained collaboration between human and machine.

This repository is the home of **Coding Commons**: an open, governed pattern language for human-AI collaboration in software development. Think of it as the field guide to going beyond vibe coding — built by practitioners, for practitioners, with the same rigor we apply to the code itself.

## The Core Idea

Flow coding is not a mode you enter after the thinking is done. It is a quality of engagement that runs through the entire arc of development — speccing, building, experimenting, debugging, reflecting.

The practice is about eliminating stuck points, not eliminating uncertainty. Expert practitioners are never truly blocked. Uncertainty becomes deliberate exploration. Complexity becomes systematic investigation. Ambiguity becomes a design opportunity. There is always a clear next action, always a pathway forward.

Each phase of development has its own conditions for flow, and its own class of stuck points. The spec eliminates direction uncertainty before the build. An experimental practice frames exploration as intentional rather than lost. A debugging methodology ensures systematic investigation rather than guessing. The craft is learning to meet the conditions for flow across the whole arc — not just during execution.

## The Commons Model

Coding Commons is governed knowledge, not a content site. Every pattern requires evidence from real development work before reaching active status. Patterns are contestable, revisable, and publicly provenance-tracked.

This is inspired by the same governance model used by [Spore](https://github.com/DarrenZal/spore) (Agent Commons) and [Intelligence Commons](https://github.com/DarrenZal/intelligence-commons) — projects developing coordination grammar and intelligence primitives for the age of agentic AI. Coding Commons is the practitioner layer: where those abstractions meet actual development work.

The learning field model (Johar, *Learning as Infrastructure*): knowledge generated by practitioners circulates back as governed patterns, not locked inside proprietary platforms. Epistemic non-enclosure is a structural commitment here, not a marketing position.
