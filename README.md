# Flow Coding

**The evolution of AI-assisted development — from vibe to craft.**

→ [flowcode.ing](https://flowcode.ing)

---

Vibe coding showed us that anyone can build with AI. Flow coding is what happens when you turn that into an art form — a practice of deep focus, creative intuition, and sustained collaboration between human and machine.

This repository is the home of **Coding Commons**: an open, governed pattern language for human-AI collaboration in software development. Think of it as the field guide to going beyond vibe coding — built by practitioners, for practitioners, with the same rigor we apply to the code itself.

## The Core Idea

Flow coding has a two-beat rhythm:

**Spec mode** — deliberate, structural. You and your AI work out what needs to exist and why. Intent becomes explicit. Unknowns surface. The output is a spec that holds direction without requiring ongoing cognitive load.

**Flow mode** — generative, fast, intuitive. The spec holds the container. Working memory commits fully to the build. Clear goals and rapid feedback create the conditions for sustained attentional depth.

The spec is not overhead. It is the pre-flight that makes flow possible — the same way a jazz musician who has internalized the chord changes can improvise freely inside them.

## The Commons Model

Coding Commons is governed knowledge, not a content site. Every pattern requires evidence from real development work before reaching active status. Patterns are contestable, revisable, and publicly provenance-tracked.

This is inspired by the same governance model used by [Spore](https://github.com/sporeproject) (Agent Commons) and [Intelligence Commons](https://github.com/intelligencecommons) — projects developing coordination grammar and intelligence primitives for the age of agentic AI. Coding Commons is the practitioner layer: where those abstractions meet actual development work.

The learning field model (Johar, *Learning as Infrastructure*): knowledge generated by practitioners circulates back as governed patterns, not locked inside proprietary platforms. Epistemic non-enclosure is a structural commitment here, not a marketing position.
